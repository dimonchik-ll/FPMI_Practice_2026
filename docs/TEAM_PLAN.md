# План параллельной разработки

## Один раз делает тимлид

1. Применяет `tower_defense_team_scaffold.patch`.
2. Импортирует ассеты через `tools/import_assets.py`.
3. Проверяет `python -m pytest -q` и `python team_launcher.py`.
4. Каждый разработчик создаёт ветку от этого коммита.

## Владение кодом

| Разработчик | Ветка | Можно менять | Нельзя менять без общего решения |
|---|---|---|---|
| 1: карта и core | `dev-core-map` | `core/`, `assets/tiles/` | `shared/`, `integration/`, чужие папки |
| 2: башни и снаряды | `dev-towers-assets` | `towers/`, `assets/towers/`, `assets/projectiles/` | `shared/`, `integration/`, чужие папки |
| 3: враги и волны | `dev-enemies-waves` | `enemies/`, `assets/enemies/` | `shared/`, `integration/`, чужие папки |
| 4: UI и экономика | `dev-ui-economy` | `ui/`, `assets/ui/`, документация UI | `shared/`, `integration/`, чужие папки |

`shared/contracts.py`, `shared/asset_manifest.py` и `integration/app.py` считаются замороженными до этапа слияния. Изменение контракта обсуждается всей командой, потому что его используют все модули.

## Контракты между модулями

- `core` отдаёт маршрут `tuple[Vector2, ...]`, клетку `GridCell` и `BuildRequest`.
- `enemies` отдаёт `tuple[EnemyView, ...]`, принимает `list[DamageCommand]`, возвращает `list[GameEvent]`.
- `towers` принимает `tuple[EnemyView, ...]`, возвращает `list[DamageCommand]`.
- `ui` получает `GameSnapshot` для отрисовки и отдаёт `UiAction` при клике.

Запрещены импорты между соседними модулями: `towers` не импортирует `enemies`, `enemies` не импортирует `towers`, UI не импортирует реализацию core.

## Задачи

### Разработчик 1 — карта и ядро

- заменить временные цвета на тайлы поля;
- улучшить карту и путь;
- добавить несколько уровней и выбор уровня;
- добавить проверку кликов и мест для строительства;
- сохранить API `GameMap` и `CoreWorld`.

Готово, когда `python -m core.sandbox` показывает карту, маршрут существует и башню нельзя поставить за пределами build slot.

### Разработчик 2 — башни и ассеты

- использовать idle/attack-спрайты арчер-башен;
- добавить снаряды и их движение;
- добавить приоритет цели и виды атак;
- добавить уровни/новые типы башен;
- не менять способ передачи урона: итог всегда `DamageCommand`.

Готово, когда `python -m towers.sandbox` показывает башню и она создаёт команды урона для `EnemyView` в радиусе.

### Разработчик 3 — враги и волны

- назначить понятные имена четырём наборам врагов;
- добавить анимации ходьбы, атаки, смерти;
- доработать логику спавна, волн и сложности;
- добавить особые свойства врагов;
- оставлять наружный интерфейс `EnemySystem` совместимым.

Готово, когда `python -m enemies.sandbox` запускает волну, враги идут по маршруту, получают урон и корректно создают `GameEvent`.

### Разработчик 4 — UI и экономика

- улучшить HUD, меню выбора башни и подсказки;
- добавить паузу, настройки, экран победы и поражения;
- балансировать стартовые деньги и стоимость;
- не помещать игровую механику башен/врагов внутрь UI;
- возвращать действия только как `UiAction`.

Готово, когда `python -m ui.sandbox` работает без карты и врагов.

## Проверки перед Pull Request

```bash
python -m pytest -q
python -m core.sandbox
python -m towers.sandbox
python -m enemies.sandbox
python -m ui.sandbox
```

В Pull Request разработчик прикладывает короткое описание, скриншот/видео и список затронутых файлов.

## Интеграция

После слияния четырёх веток тимлид проверяет `python team_launcher.py`. Если требуется поменять общий контракт, изменения выполняются отдельным Pull Request после совместного согласования.
